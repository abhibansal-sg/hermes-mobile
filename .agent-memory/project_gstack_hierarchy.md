## GStack Paperclip Hierarchy (decided 2026-04-14)

**Hermes = CEO** — Architecture truth, ruthless prioritization, standards enforcement
**Kai = COO** — Continuity, execution, operator empathy, day-to-day orchestration  
**Atlas = CTO** — Implementation authority
**Mia = PA** — Coordination (when enabled)

**Why:** Debate moderated by Claude Code with Kai and Hermes. Both made cases. Evidence from the QA audit showed GStack's primary failure mode is truth failure (config/runtime drift, silent degradation, stale surfaces, Abhi as human integration layer), not execution speed. Hermes's posture of "force the stack to stop lying to itself" is the higher-priority CEO trait for this phase.

**How to apply:** Hermes sets standards, chooses priorities, cuts dead paths, forces convergence. Kai translates standards into execution, pushes back when perfectionism creates drag. The test: in six months, is GStack both more truthful AND more operational?

**Conditions:**
- Hermes must ship, not just audit. If GStack stays in audit-loop, Kai's critique was right.
- Kai pushes back when cleanliness threatens throughput. COO is not a demotion.
- Both accepted explicitly without ego.
