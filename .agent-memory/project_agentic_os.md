Strategy-stage project at `~/projects/agentic-os` (docs/OS-VISION.md, docs/RESEARCH-2026-06.md). "Pocket Agent" reframed as an **agent operating system**: phone = control plane + trust surface; the agent runs remotely (built on the Hermes agent gateway), never on the phone. Graceful-degradation action ladder = the syscall ABI: official API → consented merchant-opt-in payment protocol → human concierge → "user taps finish". Scoped virtual cards + FaceID gates = capabilities/rings. Principle: "fails safe, never silently." Wedge = personal admin/errands (fight-my-admin, reservations, travel, shopping/returns). Build the failure path (kernel) first.

Key deep-research findings (2026-06-08, 3 fan-out runs):
- **The concierge→automation flywheel — the core thesis — has NO historical precedent.** Facebook M shut down (2018) still 70%+ human (AI never >30%); GoButler cut its human layer rather than graduating it; Adept ($414M) shipped no GA product; DoNotPay drew an FTC order. → Flywheel **demoted to a falsifiable hypothesis with a pre-committed kill-criterion**; concierge treated as a **permanent cost line** in the base case. Two caveats keep it alive: all precedents predate frontier LLMs, and the decisive modern test (do Fin/Sierra/Decagon deflection rates improve over time or plateau?) was still being researched.
- **Architecture validated**: OpenAI's ChatGPT agent independently converged on the same human-in-the-loop ladder; Princeton HAL paused its accuracy leaderboard to measure reliability (supports "the one number"); Stripe Issuing-for-agents matches the scoped-virtual-card spec (integrate, don't build).
- **Gravest risk = legal**: Amazon v. Perplexity (N.D. Cal., 2026) — CFAA violation found despite user permission after a platform C&D — threatens the **computer-use tier**; reorder the ladder so computer-use is the last resort. Merchant-opt-in protocols (ACP) are structurally authorized but have weak traction; AP2 leaves liability unresolved.

- **Theory of improvement reframed (Run 3, modern flywheel test):** autonomy ceiling moves with frontier MODEL UPGRADES (exogenous), NOT with captured-completion training (endogenous). Intercom Fin gains come from Fin 1→2→3 versions, not per-customer learning; Klarna's bot plateaued at ⅔ and the CEO reversed to hybrid in 2025. Consequence: proprietary completion data is NOT the moat (trust/distribution/official-API coverage/failure-path UX are); ride the capability curve, don't generate it; price the human handoff as a permanent premium outcome.

- **BIGGEST RISK = no data moat** (ties to flywheel finding): fast-follower or card networks (post-RBI agentic consent) replicate automation once frontier models improve; head-start is ops+network+brand, not data. Pressure-test the moat before scaling spend.

Two super-app mechanics FORECLOSED in India (bootstrap wallet, monetize payment-spread) → attempting a super app minus what made every prior one work. "Agent as aggregator" already SHIPPED by OpenAI(ACP)/Google(UCP)/Perplexity = validation + the central THREAT (model suppliers = competitors). MOAT (founder-refined, folded into OS-VISION): NOT connectors (commodity) and NOT the model (rented). The moat = **convenience that compounds** — how you interact with users + how seamlessly you eliminate the friction of accessing all these services. Precisely: convenience WINS the user, the accumulated relationship (connected graph + memory + payment-habit + trust) KEEPS them — UX is copyable in a sprint, but this user's months of context aren't. Defensible form = a **proactive inefficiency-elimination engine** (not reactive task-doer): nudges tuned to the user's life → more defensible (needs your context) + solves frequency + counter-positions frontier (they do generic global convenience; won't localize Indian daily-life frictions). Sharp rule: don't out-convenience ChatGPT in general — be radically more convenient at the *specific* affluent-Indian frictions. Reliability/trust = table stakes; concierge = one deferred tool not the headline; skill-platform = Phase 4. Still altitude + accumulated relationship, not a permanent wall → race the clock. Strong enough for a defensible vertical-concierge business, NOT to hold "super app" vs frontier incumbents. Reconciles with narrow-first (mutually reinforcing): hold platform ambition PRIVATELY, ship the wedge PUBLICLY; every dollar serves the wedge not the platform (widen early = become Tata Neu). NEW to-dos: solve the frequency problem (admin isn't daily → manufacture a hook), build per-user admin memory; RACE THE CLOCK vs OpenAI/Google/PhonePe (moat is time-boxed).

Prove the kernel on the hard case (admin), launch users on the working case (groceries+payments). Connect-to-everything-day-one = Jasper's mistake (maximizes commodity + unreliable surface at once).
