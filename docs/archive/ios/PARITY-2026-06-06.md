# Hermes Mobile ↔ Desktop feature parity (grounded census, 2026-06-06)

Source: 3-reader code census of apps/desktop + apps/shared (UI / system / protocol
clusters), full result in the session workflow output; cross-checked against the
iOS app as shipped through TestFlight build 5 (F2) + F3-H in flight.

## 1. Parity (shipped on iOS)
Streaming + thinking/reasoning disclosure + working timer · tool rows (hybrid
live→collapse) · markdown + Shiki-equivalent syntax + ANSI · edit/retry via
truncate_before_user_ordinal · interrupt · prompt queue + offline outbox ·
sessions: list/search/pin/rename/archive/delete/export, workspace grouping,
cron filter, load-more · context-window meter · model picker + reasoning
effort + personality · slash launcher (commands.catalog + slash.exec; /compress,
/usage etc. ride through it like desktop) · clarify cards · approvals
(inbox + banner) · attachments (photos/camera/scanner → /api/upload →
image.attach) · voice dictation (audio.transcribe) + read-aloud (audio.speak) ·
themes (6, as tints over native glass) · usage footer · gateway connect
(remote URL+token) — with QR pairing, which desktop lacks.

## 2. iOS-only (desktop has no equivalent)
APNs push with actionable Approve/Deny + per-decision Face ID (F2) · Live
Activities · widgets · share-sheet ingest · Siri App Intents · Spotlight ·
Face ID app lock · hermesapp:// deep links (census confirms desktop registers
NO protocol handler, has no tray, no global shortcuts) · QR pairing ·
lock-screen approvals.

## 3. Planned, census-confirmed
- F4a: @-references via complete.path + context suggestions; file browser
  (desktop's is IPC/local — iOS needs the 2 token-gated server endpoints per
  Wave 2.3 plan); subagent tree (subagent.*/delegate.* events); session.cwd.set
  + working-dir picker.
- F4b: profile switcher → target upstream's model (GET /api/profiles/sessions,
  optional profile param) per UPSTREAM-REVIEW-2026-06-06.md §4.
- Wave 4 candidates: full voice-conversation loop (desktop has it; iOS has the
  two halves), Mission Control / artifacts view.

## 4. Gaps discovered by the census (now tracked, task #7)
- FUNCTIONAL: sudo.request + secret.request events unhandled on iOS (desktop
  renders inline prompts; on mobile a sudo-gated turn would stall silently).
- Branch/checkpoint UX (branch picker, restore checkpoint, branch-in-new-chat —
  pure client work, fork = session.create with messages[] seed).
- Polish: todo checklist card, tool grouping + product/technical toggle, queue
  panel editing, KaTeX, inline audio/video + image zoom, usage analytics screen,
  prompt snippets, iPad drag-drop, personality intro copy.

## 5. Deliberately omitted
- Ratified cuts: embedded terminal/PTY (+ its add-to-chat), watch app.
- Design stance — iOS is a client/controller, not an installer/admin console:
  first-run bootstrap, provider OAuth onboarding, API-keys manager, MCP server
  management (beyond what gateway config exposes), config export/import/reset,
  auto-update (TestFlight owns it), messaging-platform setup, model-visibility
  curation, logs viewer, gateway self-update. Cheap REST adds (cron CRUD,
  skills/toolsets toggles) revisit on demand.
- No mobile meaning: window/zoom/devtools/menu-bar chrome, drag-out of file
  tree, right-rail webview preview (+ preview console/devtools), spellcheck,
  relocate-to-Applications.
